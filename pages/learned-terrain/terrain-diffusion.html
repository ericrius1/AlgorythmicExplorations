Terrain Diffusion

The paper has two connected contributions. InfiniteDiffusion is the general algorithm for querying a learned infinite sample. Terrain Diffusion is the terrain system built on top of it: a learned procedural terrain generator that can be streamed like noise but trained from real geography.

The easiest way to read the paper is to keep the layers separate. One layer is the procedural API. One layer is the infinite denoising algorithm. The final layer is the terrain-specific model stack that makes Earth-like heightfields practical.

Step 1 · Terrain is not just RGBEncoding the ground

Elevation is difficult data. A model has to represent flat plains, ocean floors, high mountains, and sharp local relief without letting the tallest regions dominate training. Terrain Diffusion uses transforms designed for that numerical problem rather than treating the height map as a normal image.

The paper uses signed square-root compression to reduce extreme dynamic range, then a Laplacian-style pyramid: a blurred low-frequency terrain layer plus residual layers that store what the blur removed. That lets broad geography and local detail be modeled at different scales. The point is not only visual quality. It is stability: the model needs to preserve both planetary structure and local terrain texture. 

raw elevation
  -> compressed elevation
  -> coarse low-frequency structure
  -> residual detail
  -> decoded terrain tile

Step 2 · A hierarchy, not one modelFrom global context to local tile

Terrain Diffusion uses a coarse-to-fine hierarchy. A coarse model supplies global context. A latent diffusion model produces the main terrain structure. A fast decoder turns that structure into high-detail output. Climate variables and geographic context guide the result, so the model is not only hallucinating plausible mountains; it is learning relationships between elevation, location, and environment.

This is why InfiniteDiffusion matters. A hierarchy can give each tile wider context without requiring the whole planet to be generated at once. The user queries a local region. The method pulls only the necessary context through the cascade. 

query region
  -> coarse context window
  -> latent terrain window
  -> decoded elevation / climate tile
  -> game or visualization

The left side is the open-world query pattern: the player moves, nearby tiles are requested, and cached context stays useful. The right side is the model stack. Step east for a warm neighboring query, or teleport to force a cold cache miss. The point is not the exact toy timing; it is the difference between first-tile cost and steady movement through already-nearby context.

Step 3 · The reported resultQuality and latency

The paper reports that naive tiling performs far worse than the learned infinite method. In the terrain FID comparison, Perlin-style blending and naive tiles are distant from the model distribution, while InfiniteDiffusion with two blending steps is much closer to the base consistency model.

The practical numbers are the more important reading. The highest quality configuration reports about 1.72 seconds to the first 512 by 512 tile and about 0.66 seconds to a neighboring second tile on the tested setup. That second number is the open-world number: once the player is moving through nearby space, cached context makes the next query cheaper.

Aside · Why the Minecraft demo matters

The Minecraft integration is not just a cute renderer. It tests the actual contract: the game asks for terrain as it streams the world. The generator must answer local queries fast enough, consistently enough, and without visible tile seams.

Step 4 · What it does not solveLimitations

InfiniteDiffusion is not free procedural noise. It is still a neural generator running over windows. The paper calls out three practical limits. Ambiguous prompts can need more blending steps to avoid artifacts. Very small scattered point queries are inefficient when they miss the cache, because the method wants to compute windows, not single coordinates. And the coarsest training layer is still a data problem: global structure has to come from somewhere.

Those limits are useful because they show where the abstraction is real. Terrain Diffusion is strongest when the caller asks for contiguous regions, as games and map viewers usually do. It is weaker when the caller asks for unrelated pinpricks across the planet.

Final explanationThe whole paper in one pass

InfiniteDiffusion takes the overlapping-window idea from MultiDiffusion and makes the target image implicit and unbounded. Instead of generating a whole world, it lazily computes only the denoising windows needed for a requested region, caching intermediate work. Because the seed fixes the underlying noise field and each coordinate maps to the same window computations, overlapping queries agree no matter what order they are requested in.

Terrain Diffusion applies that algorithm to Earth-scale terrain. It uses terrain-specific encodings, a coarse-to-fine hierarchy, and fast consistency decoding so elevation and climate tiles can be queried like procedural noise while preserving learned geographic structure. That is the bridge the title promises: learned fidelity with procedural utility.
