Only Compute the Window You Need

InfiniteDiffusion changes the question from "how do we generate an infinite image?" to "which finite windows affect the finite region we asked for?" That sounds modest, but it is the entire trick. Infinity becomes an address space, not a workload.

Think of the output as an implicit tensor with no stored boundary. The tensor has values everywhere, but values are materialized only when a query asks for them. The model is still finite. The windows are still finite. The visible region is finite. Only the coordinate system keeps going.

Step 1 · Region firstThe query is the unit of work

Let R be the region the caller wants: maybe a terrain tile, maybe a strip of chunks, maybe a camera-centered patch. InfiniteDiffusion finds every denoising window whose influence intersects R. Windows outside that influence cone do not matter and are not computed. 

query(seed, R):
  windows = windows_that_overlap_the_query(R)

  for denoising step t:
    for W in windows:
      update W using the diffusion model
    blend the window updates over R

  return R

The region can be anywhere. The cost for a fixed-size region stays bounded because only a bounded neighborhood of windows can contribute.

Drag the requested region. The domain drawn behind it can extend without limit, but only the highlighted windows affect the current answer. The lower plot is a blended output over the query slice. Moving the query changes which windows are materialized, not the definition of the world.

Step 2 · The infinite sum is localBlending without a global canvas

MultiDiffusion blends overlapping window predictions with a numerator and denominator. On an infinite canvas, the formal sum ranges over infinitely many windows, but almost all of them have zero weight at a particular pixel. InfiniteDiffusion evaluates only the nonzero terms. 

value_at_pixel =
  sum(weight_i * prediction_i) / sum(weight_i)

// In practice:
// ignore every window whose weight is zero at that pixel.

This is the same move procedural noise has always made. A point in gradient noise is affected by a small number of lattice corners. A point in InfiniteDiffusion is affected by a small number of overlapping model windows. The neighborhood is larger and learned, but the locality is the reason the method can be queried.

Step 3 · The seed fixes the worldConsistency and order independence

A procedural generator cannot change its mind because the player visited the east chunk first. InfiniteDiffusion preserves this by tying the underlying noise field to the seed and coordinates. If two region queries overlap, their shared pixels refer back to the same implicit denoising state.

That gives the method its procedural guarantees: 

same seed + same coordinate -> same value
fixed-size region           -> bounded work
independent windows          -> parallel evaluation

The paper emphasizes these properties because they are the difference between a big image generator and an open-world generator. A giant image is still an asset. A coordinate-stable query is infrastructure.

Step 4 · Memory becomes a cacheFirst tile, second tile

The first query may need to build fresh window state. A neighboring query can reuse much of it. That is why the paper reports two different latency ideas: time to first tile and time to second tile. The first measures a cold query. The second measures interactive movement after nearby context is already cached.

The cache is not the world. It is only a speedup. If the cache evicts a window, the same seed and coordinate can recreate it. That is exactly how chunk streaming works in hand-written terrain engines, but now the chunk contents are produced by a learned denoising process.

Aside · Why this is not just bigger tiling

A tiler stores or generates adjacent images and tries to hide the edges. InfiniteDiffusion defines a single implicit sample and lets each query return a slice of it. The visible tile is a view into the process, not the process itself.

CodaLearned procedural noise

This is why the paper frames InfiniteDiffusion as a bridge between learned fidelity and procedural utility. It is not faster than classic noise in the microsecond sense, but it aims at the same role: a deterministic function over an unbounded domain.
