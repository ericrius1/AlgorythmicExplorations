A World Is a Function

Alexander Goslin introduces InfiniteDiffusion as a way to make diffusion models behave more like procedural generators: not a single finished image, but something you can query forever. Before the paper's neural machinery makes sense, the old procedural trick has to be clear: an infinite world is usually not stored anywhere. It is a function.

A terrain generator takes a seed and a coordinate and returns an answer. Ask for the height at (x, z), the biome at the same point, or the tree placement in a nearby chunk, and the world recomputes that answer from arithmetic. There is no complete map hiding behind the curtain. 

height = terrain(seed, x, z)
biome  = climate(seed, x, z)
tree   = placement(seed, chunk_x, chunk_z)

Step 1 · The contractProcedural terrain is an API

A procedural world promises three things. First, seed consistency: the same seed and coordinate return the same result tomorrow. Second, random access: the engine can jump to a faraway chunk without generating every chunk in between. Third, unbounded extent: there is no natural edge, because the coordinate system keeps going.

These properties are more important than any individual noise function. They are why a game can discard a chunk behind the player and recreate it later without anyone noticing. They are why multiplayer clients agree on where the same hill is. They are why a world can be bigger than memory.

Step 2 · Random, but repeatableHashing the coordinates

White noise is easy: give every point an independent random value. It is also useless for terrain, because neighboring points have no relationship. The procedural move is to put repeatable randomness on a lattice. A hash turns an integer coordinate into a pseudorandom gradient or value, then interpolation makes the space between lattice points smooth. 

g00 = gradient(hash(seed, ix,     iz))
g10 = gradient(hash(seed, ix + 1, iz))
g01 = gradient(hash(seed, ix,     iz + 1))
g11 = gradient(hash(seed, ix + 1, iz + 1))

height = smooth_blend(g00, g10, g01, g11)

This is the small miracle behind classic terrain. The mountain is random in the statistical sense and fixed in the engineering sense. The world does not need storage because the hash is the storage.

Step 3 · Scale entersFractal noise

One layer of noise gives rolling hills. Real terrain has structure at many scales: continents, ranges, ridges, gullies, and surface roughness. So procedural terrain stacks noise at rising frequency and falling amplitude. The result is fractional Brownian motion, or fBm. 

sum = 0
amp = 1
freq = 1

for octave in octaves:
  sum += amp * noise(seed, freq * x, freq * z)
  freq *= lacunarity
  amp  *= persistence

This already gives a usable landscape. Fold the noise around zero and ridges appear. Warp the input coordinates with another noise field and the terrain starts to flow. The Ground Truth series builds exactly that kind of world from hand-written functions.

Step 4 · The failure modeNoise does not know geography

Procedural noise is fast, infinite, and controllable, but it does not understand landscapes. It approximates terrain as a texture. It can make roughness at many scales, but it does not naturally learn that mountain ranges organize valleys, valleys organize river basins, and climate variables correlate with elevation.

That is the opening tension in Goslin's paper. Procedural noise has the right utility. Learned generative models have the right fidelity. The hard problem is keeping the procedural contract while replacing the hand-written texture with a model trained from real terrain.

CodaThe target

The series target is now precise. A learned terrain generator should behave like a function: 

terrain_model(seed, region) -> elevation, climate, biome

It should be deterministic for a seed. It should answer a region query without generating the whole planet. It should let neighboring queries agree at their borders. And it should capture structure that procedural noise only hints at. The next post asks what diffusion models contribute to that bargain, and why they immediately run into the edge of the canvas.
